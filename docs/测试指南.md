# STG Reference v2 测试指南

## 1. 环境准备

### 1.1 Python 环境

需要 Python 3.10+，推荐使用已验证可用的 conda 环境：

```powershell
conda activate Z-py-3-10
```

### 1.2 安装依赖

```powershell
cd d:\after_CSU\实验室\Kan\Memory\stg_reference_v2
pip install -r requirements.txt
```

依赖项：
- `numpy>=1.23`
- `scipy>=1.10`
- `sentence-transformers>=2.6`（可选——不装则自动回退到 hashing 嵌入）
- `faiss-cpu>=1.7`（可选——不装则使用 NumPy 内积检索）
- `openai>=1.0`（仅 LLM 问答时需要）

> **注意**：如果当前环境的 PyTorch 有 DLL 问题（如 base 环境），可用 `--embedding_backend hashing` 完全绕过 sentence-transformers 和 torch。

### 1.3 降级运行（无 GPU / 无 torch）

系统支持平滑降级：
- `sentence-transformers` 不可用 → 自动使用 `hashing` 嵌入（确定性、轻量）
- `faiss` 不可用 → 使用 NumPy 内积检索

可以通过 `--embedding_backend hashing` 强制使用纯 CPU、无外部模型的模式。

### 2.2 使用原有项目数据

**是的，可以直接使用原来的数据。** 新版 `schema.py` 的归一化逻辑兼容原有数据格式：

| 原有字段 | 新版期望字段 | 兼容方式 |
|---------|------------|---------|
| `box` | `bbox` 或 `box` | ✅ `schema.py` 自动识别 `box` 字段 |
| `subject_relations` / `object_relations` | `relations` | ✅ `schema.py` 自动合并为统一的 `relations` |
| `layer_mapping` | `relations` | ✅ 自动合并 |
| 无 `frame_index`，只有 `image_path` | `frame_index` | ✅ 自动从 `image_path` 文件名中提取数字（如 `frame_000000.png` → 0） |
| `attributes` 为字符串 | `attributes` 为列表 | ✅ `normalize_attributes()` 自动处理字符串情况 |
| `label` / `tag` | `label` / `tag` | ✅ 直接兼容 |
| `score` | `score` | ✅ 直接兼容 |

### 3.3 检索测试

构建完成后，可以进行检索测试：

```powershell

python scripts/query_stg.py `
    --sample_id video_001 `
    --query "What is the player doing?" `
    --output_dir ./outputs `
    --top_k 8

# JSON 完整输出
python scripts/query_stg.py `
    --sample_id video_001 `
    --query "player shooting basketball" `
    --output_dir ./outputs `
    --top_k 8 `
    --json
```

**预期输出**：
- 事件证据列表（按 final_score 降序），每条包含事件类型、帧范围、相关实体、摘要

**如果输出 "(no event evidence above threshold)"**，说明相似度阈值过高或嵌入后端与构建时不一致。

## 4. LLM 问答测试

### 4.1 干跑模式（不调用 LLM，仅检查 prompt 和证据）

```powershell
python scripts/llm_qa.py `
    --sample_id video_001 `
    --query "What are these players doing? How many people appeared?" `
    --output_dir ./outputs `
```

这会输出完整的 JSON，包含：

- `system_prompt`：系统提示词

## 5. 新版 vs 旧版对比

| 对比项 | 旧版 (stg/) | 新版 (stg_reference_v2/) |
|-------|------------|------------------------|
| 向量存储 | 按 entity 分区的多 FAISS 索引 | 按 events/entities 两个分区，统一 VectorStore |
| 输入校验 | 无 | `schema.py` 完整校验+归一化 |
| 查询解析 | 简单子查询拆分 | `query_parser.py` 提取实体/关系/时序线索+意图推断 |
| 检索后处理 | 无重排 | 启发式 rerank（实体/关系/时序/意图加分） |
| 证据输出 | 拼接文本 | 结构化 evidence bundle（JSON + 人类可读文本） |
| LLM 集成 | 独立脚本 | `llm_adapter.py` 封装 + grounded prompt + JSON 答案格式 |
| 图导出 | 无 | `stg_graph.json`（节点+三类边） |
| 降级支持 | 无 | sentence-transformers/faiss 均可缺失 |

## 6. 常见问题排查

### Q: ModuleNotFoundError: No module named 'stg'
需要设置 `PYTHONPATH`：
```powershell
$env:PYTHONPATH = "."
```
或确保在 `stg_reference_v2` 目录下运行。

### Q: 检索结果为空
1. 确认 `--output_dir` 和 `--sample_id` 与构建时一致
2. 确认 `--embedding_backend` 与构建时一致（不同后端的向量不兼容）
3. 检查 `outputs/store/<sample_id>/` 下是否有 `.npy` 和 `_meta.json` 文件

### Q: torch DLL 加载错误
在 Windows 上某些 conda 环境的 torch 可能有 DLL 问题。解决方案：
- 使用 `conda activate Z-py-3-10`（已验证可用的环境）
- 或使用 `--embedding_backend hashing` 完全绕过 torch

### Q: 构建时一直显示 SceneGraphValidationError
检查输入 JSON 格式是否满足要求：
- 每个 object 必须有 `tag`、`label`、`bbox`（或 `box`）字段
- `bbox` 必须是长度为 4 的数组，且 x2 >= x1, y2 >= y1
- `score` 必须是数字
